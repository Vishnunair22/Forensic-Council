# Agent Capabilities & Methodologies

The Forensic Council agents utilize a hybrid approach: Large Language Models handle the cognitive ReAct loop (orchestration/deduction), while specialized machine learning scripts perform the mathematical anomaly detection.

## Confidence Scoring Methodology

Every finding generated by an agent receives a numerical `confidence_score` (0.0 to 1.0). 
1. **Mathematical Baseline**: The subprocess scripts return a raw anomaly score (e.g., `-0.8` from an `IsolationForest`).
2. **LLM Calibration**: The agent's LLM interprets this score against the context (e.g., "The image is highly compressed, so a minor ELA anomaly is expected").
3. **Arbiter Adjustment**: The Council Arbiter may demote a score if a finding is contested by another agent without sufficient proof.

## 1. Agent 1: Image Forensics

**Primary Role**: Detect pixel-level manipulation, splicing, and generative artifacts.

*   **Underlying ML Tools**:
    *   `ela_anomaly_classifier.py` (IsolationForest): Error Level Analysis.
    *   `splicing_detector.py` (GaussianMixture): Quantization inconsistency detection.
    *   `noise_fingerprint.py`: Sensor noise (PRNU) tracking.
*   **What it CAN detect**:
    *   Items "photoshopped" or spliced into an image from a different source.
    *   AI-generated (GAN/Diffusion) images via frequency analysis.
    *   Localized cloning/healing brush usage.
*   **What it CANNOT detect**:
    *   Images that have been downscaled, heavily heavily compressed (JPEG quality < 50), and printed/rescanned (this obliterates the high-frequency data required for ELA/PRNU).

## 2. Agent 2: Audio & Multimedia

**Primary Role**: Identify audio deepfakes, synthetic voices, and audio splicing.

*   **Underlying ML Tools**:
    *   `audio_splice_detector.py` (IsolationForest via `librosa` MFCCs).
*   **What it CAN detect**:
    *   Sudden dropouts or shifts in background room tone (indicating a splice).
    *   AI-generated voices lacking natural respiratory micro-pauses or exhibiting frequency-domain artifacting.
*   **What it CANNOT detect**:
    *   Mimicry by a professional human impersonator.
    *   Tampering in audio files recorded in extremely noisy environments with a signal-to-noise ratio below 10dB.

## 3. Agent 3: Object Detection

**Primary Role**: Contextual validation of items within the scene.

*   **Underlying ML Tools**:
    *   Relies on underlying image splicing scripts (`splicing_detector.py`) bounded to specific bounding boxes.
*   **What it CAN detect**:
    *   Lighting/shadow directional inconsistencies across different objects in the same frame.
    *   Scale/perspective anomalies.
*   **What it CANNOT detect**:
    *   Perfectly rendered 3D objects inserted with matched environmental HDRI lighting.

## 4. Agent 4: Video Forensics

**Primary Role**: Analyze temporal inconsistency in moving images.

*   **Underlying ML Tools**:
    *   `deepfake_frequency.py` applied to extracted intra-frames.
*   **What it CAN detect**:
    *   Face-swapping (Deepfakes) by tracking facial-boundary blending frame-over-frame.
    *   Missing frames (dropped to speed up an event) via timestamp/motion-vector mismatch.
*   **What it CANNOT detect**:
    *   Extremely low framerate (e.g., 2 FPS CCTV) footage tampering, as there is insufficient temporal data to establish a motion baseline.

## 5. Agent 5: Metadata Forensics

**Primary Role**: Structural and contextual validation of the file container.

*   **Underlying ML Tools**:
    *   `metadata_anomaly_scorer.py` (IsolationForest mapping missing/expected EXIF structures).
*   **What it CAN detect**:
    *   Software signatures (e.g., Photoshop or Premiere Pro saving the file instead of a camera firmware).
    *   GPS locations attached to the file that contradict the visible weather/lighting in the image.
    *   Mismatched creation vs modification dates.
*   **What it CANNOT detect**:
    *   Metadata that has been maliciously and perfectly scrubbed and re-written using specialized tools (like `exiftool`) if the attacker knows exactly what the target camera firmware outputs.
