## v1 Requirements

### Structural

- [x] **STRUCT-01**: All monolithic files (>1000 lines with mixed concerns) are decomposed into focused modules
- [x] **STRUCT-02**: `investigation.py` route handler extracted — `run_investigation_task` and `instrumented_run` moved to `orchestration/investigation_runner.py`
- [x] **STRUCT-03**: `react_loop.py` tool interpreters extracted to `core/tool_interpreters.py`; HITL logic extracted to `core/hitl.py`
- [x] **STRUCT-04**: `arbiter.py` split into verdict computation (`arbiter_verdict.py`), narrative synthesis (`arbiter_narrative.py`), and thin coordinator
- [x] **STRUCT-05**: `base_agent.py` self-reflection logic extracted to `agents/reflection.py`
- [x] **STRUCT-06**: Out-of-order definitions fixed (`_EXACT_MIME_EXT_MAP` and similar)
- [x] **STRUCT-07**: Import conventions enforced throughout codebase (no legacy `infra.persistence` imports)
- [x] **STRUCT-08**: All circular imports or import-time side effects resolved

### Functional

- [x] **FUNC-01**: JWT `aud` claim validation enabled — `verify_aud: False` removed from `auth.py`
- [x] **FUNC-02**: JWT algorithm upgraded to RS256 (asymmetric signing)
- [x] **FUNC-03**: Evidence cleanup pipeline functions correctly end-to-end
- [x] **FUNC-04**: All broken API endpoints identified and fixed
- [x] **FUNC-05**: WebSocket heartbeat and reconnection logic functions correctly
- [x] **FUNC-06**: HITL checkpoint pause/resume flow works end-to-end
- [x] **FUNC-07**: Chain of custody ECDSA signing produces valid, verifiable signatures on all events
- [x] **FUNC-08**: Rate limiting implemented at proxy or application layer

### Quality

- [ ] **QUAL-01**: All Python files use strict type hints (no untyped function signatures)
- [ ] **QUAL-02**: All significant actions use structured logging via `get_logger(__name__)`
- [ ] **QUAL-03**: Error handling covers all system boundaries (external APIs, DB, file I/O)
- [ ] **QUAL-04**: No dead code, commented-out blocks, or unused imports
- [ ] **QUAL-05**: Frontend components follow `use client` / `use server` conventions correctly
- [ ] **QUAL-06**: All React hooks follow hooks rules (deps arrays complete, no stale closures)
- [ ] **QUAL-07**: Test coverage meets minimum thresholds for critical paths (agents, auth, custody)
- [ ] **QUAL-08**: CI pipeline runs lint, typecheck, and unit tests on every PR

### Improvements

- [ ] **IMPV-01**: Redis working memory usage optimized (TTL policies, key namespacing)
- [ ] **IMPV-02**: PostgreSQL queries reviewed and N+1 patterns eliminated
- [ ] **IMPV-03**: Async patterns reviewed — blocking calls in async context identified and fixed
- [ ] **IMPV-04**: Frontend bundle size analyzed and optimized (code splitting, lazy loading)
- [ ] **IMPV-05**: Observability improved — structured traces for full agent pipeline execution
- [ ] **IMPV-06**: Developer experience improved (better error messages, dev tooling, local setup)
