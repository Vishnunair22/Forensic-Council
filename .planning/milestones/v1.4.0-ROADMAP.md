## Phase Details

### Phase 1: Structural Audit & Fix

**Goal:** Decompose all monolithic files and fix structural issues so each module has a single, clear responsibility.

**Requirements:** STRUCT-01, STRUCT-02, STRUCT-03, STRUCT-04, STRUCT-05, STRUCT-06, STRUCT-07, STRUCT-08

**UI hint:** no

**Success criteria:**
1. No Python file in `apps/api/` exceeds 800 lines (measured by `wc -l`)
2. `investigation.py` route handler is under 200 lines; orchestration logic lives in a separate module
3. `react_loop.py` tool interpreters and HITL logic are in separate files
4. `arbiter.py` is a thin coordinator under 400 lines
5. All imports use canonical paths — no `infra.persistence.*` references remain
6. All out-of-order definitions fixed — `grep -n "_EXACT_MIME_EXT_MAP"` shows definition before usage

**Depends on:** none

---

### Phase 2: Functional Audit & Fix

**Goal:** Identify and fix all bugs and functional gaps that break or degrade application behavior.

**Requirements:** FUNC-01, FUNC-02, FUNC-03, FUNC-04, FUNC-05, FUNC-06, FUNC-07, FUNC-08

**UI hint:** no

**Success criteria:**
1. JWT `aud` validation enabled — `auth.py` no longer contains `verify_aud: False`
2. JWT algorithm is RS256 — `config.py` `jwt_algorithm` default is `"RS256"`
3. Evidence cleanup runs without errors — no orphaned evidence files after cleanup cycle
4. All API endpoints return expected status codes under normal operation
5. HITL pause/resume flow completes end-to-end without WebSocket disconnection

**Depends on:** Phase 1

---

### Phase 3: Quality Audit & Cleanup

**Goal:** Enforce coding conventions, improve test coverage, and eliminate dead code and anti-patterns.

**Requirements:** QUAL-01, QUAL-02, QUAL-03, QUAL-04, QUAL-05, QUAL-06, QUAL-07, QUAL-08

**UI hint:** no

**Success criteria:**
1. `uv run pyright .` exits 0 with no type errors
2. `uv run ruff check .` exits 0 with no lint violations
3. `npm run type-check` exits 0
4. No unused imports — `ruff check --select F401` reports 0 violations
5. Test suite covers all agent `investigate()` entry points and auth flow

**Depends on:** Phase 2

---

### Phase 4: Improvement Audit & Implementation

**Goal:** Identify all improvement opportunities across performance, observability, and developer experience, and implement them.

**Requirements:** IMPV-01, IMPV-02, IMPV-03, IMPV-04, IMPV-05, IMPV-06

**UI hint:** no

**Success criteria:**
1. Redis keys have TTL policies — no unbounded keys remain
2. No N+1 query patterns in PostgreSQL — all list queries use joins or batch fetches
3. No blocking calls inside async functions — `asyncio` event loop never blocked
4. Frontend bundle analyzed — largest chunks identified and code-split where viable
5. Agent pipeline produces structured trace output consumable by observability tooling

**Depends on:** Phase 3
